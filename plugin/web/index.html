Satellite project — no router needed

              

                This project shares account  with another project that runs the router (the hub). The hub dispatches incoming messages here based on the binding configured on its dispatcher. You don't need to start a router for THIS project — it would conflict with the hub's wacli store lock.
              

            

          

        


        
        

          

            
            

              
Terminal-extension mode

              

                WhatsApp is a remote input/output for THIS project's claude session. Behavior is direct (no batching/typing/humanlike delay) and persona = your project's CLAUDE.md (not bot persona templates).
                The Persona / Playbooks / per-contact-Memory tabs are unused in this mode.
                You can convert to bot mode from Settings if you change your mind.

WhatsApp connection unstable

            

              Saw  disconnect/reconnect cycles in the last 60s.
              WhatsApp's protocol allows only ONE active linked-device socket per account — if you also have WhatsApp Desktop or WhatsApp Web open for the same number (), they're fighting for the socket and the bot will miss messages.
            

            

              Fix: quit WhatsApp Desktop / close all WhatsApp Web tabs, then .

Playbooks — not used in terminal-extension mode

        

          

            Playbooks are tag-driven interaction guides (new-stranger / friend / lead / ...) — useful when the bot talks to many people. In terminal-extension mode you're the only one messaging, so relationship classification doesn't apply.
          

          

            If you convert to bot mode (Settings), playbooks become available.
          

        

      


      
      

        

          
Relationship playbooks

          
          

            

              Tag-driven interaction guides (e.g., new-stranger, friend, lead). Each contact's card.md has a relationship_tag; agent reads the matching playbook on each turn.
            

            
          

        

        

          

            
            
          

          
        

      


      
      

        
Settings

        
Project metadata + danger zone.


        

          
        


        
        
Project mode

        

          

            bot: full chatbot — humanlike batching/delays, persona templates, per-contact memory v2, playbooks. Use when bot talks to many people.


            terminal-extension: lean — WhatsApp is just an extra input channel for your project's claude. Direct replies, no batching, no persona templates. Use when YOU are the only one messaging.
          

          

            
            
          

        


        
        
Owner JID — share session with terminal

        

          

            Designate one JID as "owner". WhatsApp messages from that JID use the SAME session UUID as your project's terminal claude → both share conversation state. When you run claude in this dir, the WA chat continues that conversation. Concurrent writes serialized via lock check (router will defer if you're actively running claude in terminal).
