# cc-whatsapp plugin

A Claude Code plugin that binds a WhatsApp number to a project.

## Install (Phase 1: local dev)

```sh
# Build the server binary once
cd ~/Projects/cc-whatsapp && make build

# In any consumer project:
cd ~/your-project
claude --plugin-dir ~/Projects/cc-whatsapp/plugin --dangerously-skip-permissions
```

## Use

Inside Claude Code (in your project):

```
/cc-whatsapp:init     # configure (account name, persona templates, dirs)
/cc-whatsapp:pair     # scan QR to link a WhatsApp number to this project
/cc-whatsapp:allow <jid>   # whitelist who can message the bot
/cc-whatsapp:start    # launch router in a visible Terminal (bot online)
/cc-whatsapp:status   # check state
/cc-whatsapp:stop     # take bot offline
```

## What lives where

```
<your-project>/.claude/cc-whatsapp/
├── config.json     # which wacli account; project metadata
├── access.json     # JID allowlist for this bot
├── sessions.json   # per-JID claude session UUIDs (memory continuity)
├── .secret         # HMAC for webhook auth
├── agent/          # persona — edit these to give the bot personality
│   ├── IDENTITY.md  SOUL.md  STYLE.md  AGENTS.md  MEMORY.md
│   └── contacts/   # per-contact long-term memory (.md per JID)
├── inbox/          # incoming media downloads
├── trace.log       # router event log
└── run.command     # generated by /start, opens Terminal for live router
```

## Env vars (rarely needed)

- `CC_WHATSAPP_PROJECT_DIR` — override `.claude/cc-whatsapp/` location
- `CC_WHATSAPP_BIN` — path to the server binary
- `CC_WHATSAPP_CHAT_MODEL` — default `claude-haiku-4-5-20251001`
- `CC_WHATSAPP_COLLECT_WINDOW_MS` — default 60000 (humanlike batching)
- `CC_WHATSAPP_PRE_REPLY_MIN_MS` / `_MAX_MS` — default 30000-60000
- `CC_WHATSAPP_DRY_RUN=1` — state machine runs without invoking claude/wacli
